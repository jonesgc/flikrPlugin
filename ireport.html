Plugin Overview

As of writing this report the plugin has the ability to display a single photo or multiple photos. These photos are retrieved from flikr after the search query has been processed, this query is made of arguments of the users choosing, due to the way the plugin was designed the only argument that will always be passed to the flikr sever is the woeID of the city. The cities that I have submitted are the twins we selected for the group assignment, Birmingham and Chicago. However new cities could easily be added to the api_config.json. The other arguments are optional and the plugin will still function if they are not used.

I had intended to implement a version of my plugin that would link to the group assignment. The aim of this was to search Places of interest (POI) in flikr. However flikr requires a flikr place id, this can be retireved with another API method which then needs to be searched again. In the end I did not have enough time to implement this feature. The display multiple images feature is the feature I am least pround of. Since it does not work the way I wanted it to. Originally I had imaged this feature would add the ability to show multiple images in "pages" then the user could scroll the pages. And that this functionaility would be provided on an asynchronous basis like the display single method. But due to problems with the asynch requests I had to change this method to be synchronos in order to display each image correctly.
